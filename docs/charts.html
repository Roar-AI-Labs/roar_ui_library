Charts & data visualization

These examples use Chart.js (MIT): real data, native tooltips on hover, and legends that match Roar tokens via chart-theme.js. Line and area series use soft vertical gradients under the stroke; point markers stay hidden until you hover (larger hit targets, no dot clutter). Colors follow the accent palette you pick in the toolbar. Two densities share the same .rs-chart shell: full (.rs-chart--featured) for reports and detail, and dashboard panels (.rs-chart--panel) — shorter canvas, tighter type, still fully interactive (hover for metrics, same interaction modes).

Beyond Chart.js

Chart.js covers most product dashboards (cartesian, radial, bubble, mixed axes). For Sankey, geo / choropleth, heavy financial OHLC, or zoomable million-point series, pair the same .rs-chart shell with Apache ECharts, Vega-Lite, or Observable Plot, and reuse rsChartTokens() for colors.

Type checklist

What Chart.js handles out of the box vs. when to reach for another library.

  • Line, area, step — Chart.js ✓ (gallery above).
  • Bar / stacked / horizontal / histogram-style — Chart.js ✓.
  • Mixed bar + line (dual axis) — Chart.js ✓.
  • Doughnut / pie / polar / radar — Chart.js ✓.
  • Bubble / scatter — Chart.js ✓.
  • Waterfall, funnel, candlestick, heatmap, Sankey, map — plugins or ECharts / Vega / D3.
  • Full vs dashboard panel — same library; only shell class and canvas height change.
